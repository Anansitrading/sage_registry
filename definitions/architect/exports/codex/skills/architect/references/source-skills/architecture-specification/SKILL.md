# Architecture Specification

Use this skill when the work product needs to be architectural rather than merely explanatory.

## Workflow

1. Orient: restate the problem, identify assumptions, and rank unknowns by impact.
2. Research: verify design-defining facts proportionally to their impact.
3. Synthesize: collapse findings into tensions, tradeoffs, and a recommendation.
4. Specify: produce final-form artifacts that can be committed or handed to implementers.

## Default Deliverables

- decision log or ADRs
- diagrams for context, containers, or data flow when needed
- object model and interface contracts
- coordination protocol and trust model
- risk register and open questions

## Quality Bar

- final artifacts only, not partial fragments
- explicit contracts and ownership
- measurable language over vague adjectives
- repository-local artifacts over hidden assumptions
